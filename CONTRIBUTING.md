# Contributing to Uni Safar 🤝

We love your input! We want to make contributing to **Uni Safar** as easy and transparent as possible, whether it's:

*   Reporting a bug
*   Discussing the current state of the code
*   Submitting a fix
*   Proposing new features

## 📋 Code of Conduct
Please be respectful and constructive in all communication with other contributors.

## 🚀 How to Contribute

### 1. Report Bugs / Propose Features
If you find a bug or want to suggest an improvement, please open a GitHub Issue in the repository:
*   Use a clear and descriptive title.
*   Describe the steps to reproduce the bug.
*   Provide context about your device (Android, iOS) and system versions.

### 2. Submit a Pull Request (PR)
If you want to write code and fix issues:
1.  **Fork the Repository** and clone it locally.
2.  **Create a Branch**:
    ```bash
    git checkout -b feature/your-feature-name
    # or
    git checkout -b fix/bug-description
    ```
3.  **Implement your changes**: Make sure to test your code locally on physical devices or emulators before committing.
4.  **Format and Clean Code**: Check that variables are well-named, and no raw credentials or personal tokens are committed.
5.  **Commit your changes**:
    ```bash
    git commit -m "Add feature/fix description"
    ```
6.  **Push your branch**:
    ```bash
    git push origin feature/your-feature-name
    ```
7.  **Submit a Pull Request** to the main branch of the parent repository.

### 🔍 Pull Request Guidelines
*   Write clear commit messages.
*   Reference any related Issues in the PR description (e.g., `Closes #12`).
*   Keep changes focused. Avoid bundling unrelated fixes into a single PR.

---

Thank you for helping make **Uni Safar** better for students! 🚀
